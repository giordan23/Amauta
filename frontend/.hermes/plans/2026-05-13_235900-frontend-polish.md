# Plan: Pulido Frontend AMAUTA (Mobile + Desktop)

## Contexto

Proyecto: `/mnt/d/dev/papa/amauta/frontend/`
Stack: Vite + React 19 + Tailwind CSS 4 + React Router v7
El login está "decente" según Giordan. El resto necesita revisión.
Dependencias: NO tocar.

## Objetivo

Pulir todas las páginas del frontend (mobile + desktop) con énfasis en:
- Consistencia visual mobile/desktop
- Mejora de mantenibilidad (componentes compartidos, LayoutWrapper)
- Navegación coherente
- Preparar las páginas para integración real de datos (API)

## Riesgos y tradeoffs

1. **Tiempo:** Son 6 páginas. Implementar una a una puede tomar varias sesiones.
2. **Responsive:** El CSS actual tiene mobile/desktop separados. Hay que mantener ambos.
3. **API mocks:** Mientras backend no tenga endpoints, las pages mostrarán datos de ejemplo. Diseñar para que sea fácil conectar luego.
4. **Chart library:** No incluir charting en este PR — es scope creep. Dejar placeholder.

## Open questions — RESUELTOS

1. **Chart de progreso:** Puede esperar — incluir placeholder "próximamente" de nuevo
2. **Endpoints API:** No existen aún. El objetivo de este PR es UI/UX profesional y mantenible. Las pages usarán datos mock estáticos pero con estructura lista para conectar API después (hooks de React Query ya configurados en authStore).

## Alcance de este PR

Solo UI/UX y mantenibilidad:
- Layout system compartido (AppLayout, Navbar)
- Component library (Card, StatCard, FilterBar)
- Consistencia visual mobile + desktop
- Código mantenible y extensible
- Lists, filtros, forms funcionales en frontend (sin API real)

**NO incluir:** conexión a API real, charting, backend changes.
