# AMAUTA — Project Context

## Purpose
Exam simulator for Peruvian university admissions (UNMSM, UNI, UNSAAC). Students practice with real exam questions, get immediate feedback, and track progress.

**Language:** Spanish (Peru)
**Project root:** /mnt/d/dev/papa/amauta

## MVP Scope (Fase 1)
- Registration/Login (email + password)
- Onboarding: choose country + target university
- Admin panel to upload questions manually
- Question bank (university, period, area, career, difficulty)
- Configurable exam simulation
- Results screen (score, correct/wrong answers)

**Excluded:** IA features, OAuth real, bulk CSV upload, student question submission, advanced stats, monetization

## Stack

### Backend
- Node.js 20 + Express 5 + TypeScript strict
- Prisma ORM + PostgreSQL (Supabase)
- Supabase Auth (@supabase/supabase-js server-side)
- Zod validation
- Jest + Supertest testing
- Winston/pino logging
- **Port:** 3000
- **Base URL:** http://localhost:3000/api/v1

### Mobile
- React Native (Expo SDK 51+)
- Expo Router (file-based routing in src/app/)
- Zustand (state management)
- React Query / TanStack Query
- React Native Paper (Material Design UI)
- Expo SecureStore (token persistence)
- **Expo port:** 19006

## Response Shape Convention

**CORRECT (from agents):**
```json
{ "data": {}, "error": null }
{ "data": null, "error": { "code": "ERROR_CODE", "message": "..." } }
```
